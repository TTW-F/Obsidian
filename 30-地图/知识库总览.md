# 知识库总览

## 这是什么

这页是整个知识库的总入口。

它的作用不是重复正文，而是先回答三个问题：

- 这个库大致在记录什么
- 从哪里开始最合适
- 新内容通常应该流向哪里

## 为什么重要

- 当目录越来越多时，如果没有总览页，后续进入路径会越来越散
- 总览页能帮助我先分清收集区、领域区、主题区和地图区各自承担什么角色
- 它也是后续新主题长出来时最自然的回链入口

## 目录规则

- `00-收集`：先接住信息，再决定是否沉淀
- `10-领域`：长期持续维护的能力域
- `20-主题`：抽象后的知识主题、方法、模型
- `30-地图`：索引、路线、导航
- `90-模板`：复用模板

## 推荐工作流

1. 新输入先进 `00-收集`
2. 有长期价值的内容整理进 `10-领域` 或 `20-主题`
3. 每篇正式笔记尽量补上“我的理解”“易错点”“相关笔记”
4. 重要主题都回链到地图页

## 当前重点沉淀方向

- 命令行

- Agentic CLI / Claude Code

## 我的使用原则

- 少建目录，多做索引
- 资料型笔记要慢慢补成自己的表达
- 一篇笔记尽量只解决一个核心主题
- 允许先有草稿，再逐步重写
