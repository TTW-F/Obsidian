## 目录规则

- `00-收集`：先接住信息，再决定是否沉淀
- `10-领域`：长期持续维护的能力域
- `20-主题`：抽象后的知识主题、方法、模型
- `30-地图`：索引、路线、导航
- `90-模板`：复用模板

## 推荐工作流

1. 新输入先进 `00-收集`
2. 有长期价值的内容整理进 `10-领域` 或 `20-主题`
3. 每篇正式笔记尽量补上“我的理解”“易错点”“相关笔记”
4. 重要主题都回链到地图页

## 当前重点沉淀方向

- 命令行
- Git / GitHub
- 数据结构与算法
- Agentic CLI / Claude Code

## 维护原则

- 少建目录，多做索引
- 资料型笔记要慢慢补成自己的表达
- 一篇笔记只解决一个核心主题
