# Obsidian Map Curator

把 `30-地图` 里的笔记当作导航系统来维护，而不是当作普通资料页。

目标是让一位未来的自己能从地图笔记快速找到入口、理解结构，并顺着链接继续深入。

## 工作流

1. 明确这张地图服务的范围。
   先回答：它覆盖哪个领域、主题或项目，边界在哪里。
2. 识别核心节点。
   把最值得首先阅读的总览、基础笔记、关键方法、常见问题找出来。
3. 组织导航顺序。
   通常按“总览 -> 子主题 -> 关键笔记 -> 延伸阅读”组织，不要平铺直叙地罗列。
4. 补足缺失连接。
   如果地图里出现空洞，指出应该补哪类笔记，而不是假装结构完整。
5. 保持轻量。
   地图应该帮助导航，不应该重复正文细节。

## 地图写法

先读 `references/map-patterns.md`。

优先让地图具备这些部分：

- 这张地图覆盖什么
- 建议从哪里开始看
- 关键主题或分支
- 重要笔记入口
- 还缺什么

如果已有地图结构可用，增量修改即可，不要为了统一格式推翻全部内容。

## 链接原则

- 优先链接真正能承接阅读的笔记。
- 如果某个子主题下只有一篇薄弱笔记，可以先暴露这个缺口。
- 如果同一层级项目过多，先分组再列链接。
- 对“总览”“规范”“模板”“工作流”这类枢纽笔记给予更高优先级。

## 输出要求

完成任务时，尽量给出：

- 地图当前服务的范围定义。
- 新增或重排了哪些入口。
- 哪些链接被删掉或降级了。
- 哪些缺口值得后续补笔记。

## 边界

- 不要把地图写成百科目录。
- 不要堆太多二级以下细枝末节。
- 不要用地图替代正式知识笔记。
- 不要为每篇笔记都强行挂到地图里；入口要有层次感。
