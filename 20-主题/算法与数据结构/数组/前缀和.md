# 前缀和

## 这是什么

前缀和可以先理解成：先做一遍预处理，把“从开头到当前位置的累计结果”存下来，后面就能更快地回答区间问题。

在最常见的数组场景里，它记录的是累计和，但这个思路本身并不限于求和。

最核心的价值是：

把很多重复的区间计算，变成一次预处理加上很多次快速查询。

## 为什么重要

- 很多区间和问题如果每次都重新遍历，会产生大量重复计算
- 前缀和是很多更复杂技巧的基础，比如差分、二维前缀和、哈希加前缀和
- 它能把“连续区间”这个结构用非常稳定的方式表达出来

## 最核心的公式

如果 `prefix[i]` 表示前 `i` 个元素的和，那么区间 `[l, r]` 的和可以写成：

`prefix[r + 1] - prefix[l]`

我更喜欢把它理解成：

- 大区间减去前面那一段
- 剩下的就是目标区间

这个理解比死背公式更稳。

## 为什么经常写成 `n + 1` 长度

很多实现会让 `prefix[0] = 0`，然后：

- `prefix[1]` 表示前 1 个元素的和
- `prefix[2]` 表示前 2 个元素的和

这样写的好处是区间公式统一，不需要反复讨论 `l = 0` 的特殊情况。

```python
prefix = [0] * (len(nums) + 1)
for i in range(len(nums)):
    prefix[i + 1] = prefix[i] + nums[i]
```

## 一个最常见的例子

如果数组是：

`[2, 4, 1, 3]`

那么前缀和数组可以是：

`[0, 2, 6, 7, 10]`

这时区间 `[1, 3]` 的和，也就是 `4 + 1 + 3`，可以直接算成：

`prefix[4] - prefix[1] = 10 - 2 = 8`

这个例子最能说明前缀和的本质：

不是更神秘的算法，而是把重复区间计算提前做掉。

## 它适合什么问题

前缀和通常适合这些场景：

- 多次查询区间和
- 题目本质是连续区间
- 允许先做一次预处理，再反复回答问题

如果只是查一次总和，直接遍历可能更简单；但如果会频繁查区间，前缀和就非常值。

## 前缀和和滑动窗口的区别

这两者都和连续区间有关，但擅长的方向不一样。

我现在会这样区分：

- 前缀和更擅长“快速回答某个区间值是多少”
- 滑动窗口更擅长“动态维护一个满足条件的区间”

所以如果题目核心是大量区间查询，我优先想前缀和；如果核心是边走边维护约束，我优先想滑动窗口。

## 易错点

- 容易把 `prefix[i]` 理解成“到下标 `i` 为止”，结果和代码下标定义打架
- 容易在 `l = 0` 时额外开分支，说明前缀数组定义还不统一
- 容易只会套区间和公式，却不知道它解决的是“重复计算”问题
- 容易把非连续选择的问题也硬转成前缀和

## 我的理解

前缀和最值得记住的是：

它是在拿空间换区间查询时间，把重复劳动提前做掉。

一旦题目里连续区间反复出现，我就会立刻想起这个工具。
