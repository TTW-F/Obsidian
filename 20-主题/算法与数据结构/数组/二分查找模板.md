# 二分查找模板

## 这是什么

这篇笔记沉淀的是二分查找里最值得反复复用的写法和边界规则。

它的重点不是背代码，而是记住：区间定义一旦确定，循环条件和边界更新就必须始终跟着它走。

## 为什么重要

- 二分查找思路本身不难，真正容易出错的是边界
- 很多题并不是不会做，而是写到一半把 `[left, right]` 和 `[left, right)` 混了
- 把模板记成“区间不变量”之后，后面的左边界、右边界、答案区间二分都会更稳

## 使用前提

二分查找通常适合下面这类场景：

- 数据有序
- 可以明确判断答案在左边还是右边
- 区间能随着判断不断缩小

如果题目不满足这些条件，就不要机械套二分。

## 模板一：左闭右闭 `[left, right]`

这是最适合先记住的一版。

```cpp
int binarySearch(vector<int>& nums, int target) {
    int left = 0;
    int right = nums.size() - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] == target) {
            return mid;
        }

        if (nums[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    return -1;
}
```

这一版最该记住的三个点：

- `while (left <= right)`
- 去右边时 `left = mid + 1`
- 去左边时 `right = mid - 1`

## 模板二：左闭右开 `[left, right)`

```cpp
int binarySearch(vector<int>& nums, int target) {
    int left = 0;
    int right = nums.size();

    while (left < right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] == target) {
            return mid;
        }

        if (nums[mid] < target) {
            left = mid + 1;
        } else {
            right = mid;
        }
    }

    return -1;
}
```

这一版最该记住的是：

- `while (left < right)`
- 去左边时 `right = mid`

因为 `right` 本来就是开区间端点，所以不能再减一。

## 真正该记住的不是代码，而是区间规则

二分查找最重要的一句话是：

区间定义一旦确定，后面的循环条件和边界更新都必须服务于这个定义。

也就是说：

- `[left, right]` 就坚持闭区间逻辑
- `[left, right)` 就坚持半开区间逻辑

不要混写。

## 一个最常见的错误场景

比如你明明定义的是 `[left, right]`，却写了：

```cpp
while (left < right)
```

或者：

```cpp
right = mid
```

这种错误最可怕的地方在于，代码看起来很像对的，但边界已经乱了。

## 常见变形方向

- 查找目标值是否存在
- 查找插入位置
- 查找左边界
- 查找右边界
- 在答案区间上二分

这些题型的共同点不是“长得像模板”，而是都依赖区间不变量。

## 易错点

- `while` 条件和区间定义不一致
- `right = mid` 和 `right = mid - 1` 混淆
- 忘了数组有序这个前提
- `mid` 直接写成 `(left + right) / 2`

## 我的理解

二分查找真正该记住的不是两份代码，而是一种纪律：

先定区间，再写循环，最后让每次更新都维护住这个区间定义。

一旦这一点稳了，模板只是顺手写出来的结果。
