# 二分查找模板

## 这篇笔记要解决什么

这篇专门沉淀二分查找的“可复用写法”，方便以后做题时直接调用，不必每次从头重新想边界。

## 使用前提

二分查找通常适合下面这类场景：

- 数据有序
- 可以明确判断答案在左边还是右边
- 区间能随着判断不断缩小

如果题目不满足这些条件，就不要机械套二分。

## 模板一：左闭右闭 `[left, right]`

这是最适合先记住的一版。

```cpp
int binarySearch(vector<int>& nums, int target) {
    int left = 0;
    int right = nums.size() - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] == target) {
            return mid;
        }

        if (nums[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    return -1;
}
```

要点：

- `while (left <= right)`
- 左边界更新是 `left = mid + 1`
- 右边界更新是 `right = mid - 1`

## 模板二：左闭右开 `[left, right)`

```cpp
int binarySearch(vector<int>& nums, int target) {
    int left = 0;
    int right = nums.size();

    while (left < right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] == target) {
            return mid;
        }

        if (nums[mid] < target) {
            left = mid + 1;
        } else {
            right = mid;
        }
    }

    return -1;
}
```

要点：

- `while (left < right)`
- 右边界更新是 `right = mid`

## 我真正要记住的不是代码，而是这个规则

区间定义一旦确定，后面的循环条件和边界更新方式都必须服务于这个定义。

也就是说：

- `[left, right]` 就坚持闭区间逻辑
- `[left, right)` 就坚持半开区间逻辑

不要混写。

## 常见变形方向

- 查找目标值是否存在
- 查找插入位置
- 查找左边界
- 查找右边界
- 在答案区间上二分

## 最容易写错的点

- `while` 条件和区间定义不一致
- `right = mid` 和 `right = mid - 1` 混淆
- 忘了数组有序这个前提
- `mid` 直接写成 `(left + right) / 2`
