# 数组

## 这是什么

数组是存放在连续内存空间上的同类型数据集合，可以通过下标快速访问元素。

## 为什么重要

数组是最基础的数据结构之一，很多更复杂的题型其实都是从数组延伸出来的。

如果数组这一层理解不稳，后面的双指针、滑动窗口、前缀和、二分查找都会容易学得散。

## 数组最核心的两个特点

- 下标通常从 `0` 开始
- 元素在内存中连续存放

这两个特点会直接决定它的优缺点。

## 数组的优势

### 1. 访问快

因为内存连续，所以可以通过下标直接定位到元素。

这意味着：

- `nums[0]`
- `nums[5]`
- `nums[i]`

这类访问通常都很高效。

### 2. 结构简单

数组是很多算法题最常见的输入形式，也很适合作为理解算法模型的起点。

## 数组的代价

### 1. 插入和删除不灵活

如果要在中间插入或删除元素，后面的元素往往需要整体移动。

这也是为什么数组虽然访问快，但修改中间位置的成本可能高。

### 2. 大小通常不够灵活

在很多语言里，原生数组长度固定。

补充理解：

- C / C++ 里的原生数组更接近“真正的数组”
- `vector` 这种结构底层虽然基于数组，但它本身更像容器，不等同于原生数组

## 我对“数组不能删，只能覆盖”的理解

更准确地说：

- 对原生定长数组来说，长度本身不会自动缩短
- 所谓“删除一个元素”，本质上常常是把后面的元素前移，再用逻辑长度表示现在有效的数据范围

所以题目里常说的“删除数组元素”，很多时候并不是物理删除，而是逻辑删除。

## 二维数组怎么理解

二维数组可以看成“数组里的数组”或者“按行列组织的数据结构”。

需要注意：

- 在不同语言里，二维数组的底层实现不完全一样
- C++ 中二维数组在地址空间上通常是连续的
- Java 中更像“数组对象里再装数组对象”，不强调直接暴露内存地址

## 做题时最常用的思考方式

看到数组题时，可以先问自己：

1. 题目是在考随机访问，还是在考元素移动？
2. 数据是否有序？
3. 能不能利用下标关系做文章？
4. 能不能转成双指针、滑动窗口、前缀和或二分？

## 我的理解

数组最值得记住的不是定义，而是这句话：

连续内存带来了“访问快”，也带来了“中间修改成本高”。

很多数组题，最后其实都在围绕这两个特点做文章。
