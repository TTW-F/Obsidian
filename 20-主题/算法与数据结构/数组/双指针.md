# 双指针

## 这是什么

双指针可以先理解成：不用一个下标单独扫完整个问题，而是同时维护两个位置，让它们一起移动来表达结构信息。

这两个位置不一定真的只是“左和右”。

更准确地说，双指针是在用两个游标维护下面这些关系中的一种：

- 当前有效区间
- 已处理区和未处理区
- 读入位置和写回位置
- 两端向中间收缩的边界

## 为什么重要

- 很多数组题如果只用单指针，很容易陷入重复遍历
- 双指针常常能把额外空间降下来，也能让边界更清楚
- 它不是单一题型，而是一组非常高频的移动策略

## 我现在会把双指针分成三种常见用法

### 1. 对撞指针

两个指针从两端向中间移动。

适合的问题通常有这些特点：

- 数组有序，或者可以按某种大小关系判断下一步
- 需要在两端之间做配对
- 目标和区间边界有关

典型例子：

- 有序数组找两数之和
- 判断回文串
- 盛最多水的容器

这里最关键的不是“左右各放一个指针”，而是每次移动都要有理由。

如果移动一侧不能排除一批无效解，那就说明策略还没站稳。

### 2. 快慢指针

两个指针同向移动，但速度或角色不同。

最常见的是：

- 一个负责读原数据
- 一个负责写有效结果

典型例子：

- 删除数组中的指定元素
- 原地去重
- 移动零

这类题最值得记住的一点是：很多题里的“删除”，本质上不是物理删除，而是维护有效区间。

### 3. 窗口型双指针

左右指针同向移动，用来维护一个连续区间。

它和滑动窗口非常接近。可以把滑动窗口看成“双指针里对区间约束更明确的一支”。

典型例子：

- 最长不重复子串
- 长度最小的子数组
- 连续区间和满足条件的问题

## 一个判断题目能不能用双指针的简单问题

我现在看到数组题时，会先问自己：

能不能只靠“移动边界”，就把一批不可能的情况一起排掉？

如果可以，双指针通常就值得尝试。

因为它的强项从来不是“两个下标比较酷”，而是每移动一步，都能减少搜索空间。

## 一个最常见的例子

比如删除数组中的某个值：

1. `fast` 负责读每个元素
2. `slow` 负责写保留下来的元素
3. 当 `nums[fast]` 不是目标值时，把它写到 `nums[slow]`
4. 最后返回 `slow`

```python
slow = 0
for fast in range(len(nums)):
    if nums[fast] != val:
        nums[slow] = nums[fast]
        slow += 1
```

这里最重要的理解不是代码，而是：

- `fast` 负责扫描原数组
- `slow` 负责维护“当前有效结果写到哪”

## 双指针和滑动窗口不要混着背

很多人会把双指针和滑动窗口当成两个完全分开的方法。

我现在更倾向于这样理解：

- 双指针是更大的方法族
- 滑动窗口是其中专门处理连续区间约束的一类

这样记更稳，因为不会遇到窗口题就以为是另一套完全无关的东西。

## 易错点

- 容易只记模板，不知道两个指针各自维护什么含义
- 容易移动指针却说不清“为什么这一步不会漏解”
- 容易把对撞指针、快慢指针、窗口型双指针混成一类口号
- 容易写对代码但没意识到自己维护的是有效区间

## 我的理解

双指针最值得记住的不是“两个人一起走”，而是：

用两个位置去表达结构，再靠移动边界减少搜索空间。

只要这层理解稳了，很多题就不再是背模板，而是在选合适的边界维护方式。
