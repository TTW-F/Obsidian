# DFS 与 BFS

## 这是什么

DFS 和 BFS 可以先理解成两种完全不同的搜索顺序。

- DFS 是尽量先往深处走
- BFS 是按距离或层级一层一层扩开

它们都能“遍历”，但真正的差别不在会不会走到所有节点，而在于：

你希望按什么顺序接触问题空间。

## 为什么重要

- 树、图、网格、回溯题里都会频繁碰到它们
- 很多题不是不会做，而是第一步选错了搜索顺序
- 只要把 DFS 和 BFS 的适用场景分清，很多题会很快收敛

## DFS 最该怎么理解

我现在更倾向于把 DFS 理解成：

沿着一条路尽量走到底，走不下去再退回来。

这种感觉天然适合：

- 枚举所有可能路径
- 递归定义明显的问题
- 需要深入结构内部再返回组合答案的问题

所以二叉树递归、回溯、连通块搜索常常都会长得像 DFS。

## BFS 最该怎么理解

BFS 更像：

从起点出发，把离当前最近的一圈先处理完，再处理下一圈。

这种顺序天然适合：

- 最短步数
- 最少层数
- 一层一层统计结果
- 扩散式搜索

所以层序遍历、迷宫最短路、无权图最短路径经常天然更适合 BFS。

## 一个最关键的判断问题

我现在看到搜索题时，最先会问：

题目是更关心“结构深入”，还是更关心“离起点多远”？

如果更关心深入过程、路径和撤销，通常偏 DFS。

如果更关心层级、最短步数、最先到达，通常偏 BFS。

## 为什么 BFS 常和队列绑在一起

因为 BFS 需要保证“先发现的节点先处理”，这样才能维持距离或层级顺序。

队列恰好天然满足这个要求：

- 新发现的节点排到后面
- 当前这一层的节点先被逐个取出

所以 BFS 不是“顺便用队列”，而是顺序需求正好对应队列。

## 为什么 DFS 常和递归绑在一起

因为 DFS 的核心动作是：

- 先往下一层走
- 走完再退回来

这和递归调用栈的行为天然一致。

当然 DFS 也可以手写栈来做，但递归之所以常见，不是巧合，而是结构正好贴合。

## 一个很常见的对比场景

比如二叉树最大深度和二叉树层序遍历：

- 最大深度更容易写成 DFS，因为它强调子树递归关系
- 层序遍历更自然写成 BFS，因为它强调一层一层看节点

这个对比很能提醒我：

同样都是树题，搜索顺序还是得看题目关心的对象。

## DFS 和回溯不要完全混成一件事

回溯常常会用 DFS，但两者不完全等价。

- DFS 更强调搜索顺序
- 回溯更强调做选择、撤销选择、枚举所有解

所以“所有回溯都像 DFS”，但“不是所有 DFS 都是回溯”。

## 易错点

- 容易只记“DFS 用递归，BFS 用队列”，却不知道题目为什么该这么选
- 容易把最短路径题硬写成 DFS
- 容易把所有递归遍历都叫回溯
- 容易忽略 BFS 的层级语义，只把它当成另一种遍历

## 我的理解

DFS 与 BFS 最值得记住的不是代码模板，而是：

你到底想先看深度，还是先看距离。

只要把这个顺序需求想明白，工具和写法通常都会跟着清楚起来。
