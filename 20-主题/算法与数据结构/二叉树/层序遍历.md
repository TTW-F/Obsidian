# 层序遍历

## 这是什么

层序遍历可以先理解成：

按照“从上到下、从左到右”的层级顺序，一层一层访问二叉树节点。

它和前序、中序、后序最大的不同在于，层序遍历关心的不是递归时机，而是节点所在的层级。

## 为什么重要

- 它是 BFS 在树上的最典型落点
- 很多树题虽然看起来是“遍历”，但真正要的是逐层结果
- 一旦题目关心每层节点数、每层最大值、最短层数，层序遍历通常就会变得很自然

## 它为什么天然适合队列

因为层序遍历的核心需求是：

- 先处理当前这一层
- 再处理下一层

队列刚好能维持这种顺序：

- 先进入队列的节点先被处理
- 当前节点的左右孩子会排到后面，等待下一轮处理

所以层序遍历不是“可以用队列”，而是顺序需求本身就在呼唤队列。

## 一个最基本的处理方式

最常见的写法是：

1. 根节点先入队
2. 每次取出队头节点
3. 把它的左右孩子依次加入队尾

```python
queue = deque([root])
while queue:
    node = queue.popleft()
    if node.left:
        queue.append(node.left)
    if node.right:
        queue.append(node.right)
```

这段代码最值得记住的不是形式，而是：

队列里装着“接下来按层级顺序要处理的节点”。

## 如果题目要求“按层返回结果”怎么办

这时关键不是换算法，而是把“当前层”这个边界取出来。

常见做法是：

- 每轮先记下当前队列长度
- 这个长度就代表当前层节点数
- 只处理这么多个节点，再进入下一层

我现在会把这个长度理解成：

当前这一层的天然边界。

## 一个典型场景

比如“二叉树的层序遍历”或“每层最大值”：

题目本质都不是在问“遍历所有节点”，而是在问：

这一层的节点集合到底是什么。

所以如果题目开始强调“每层”，我就会立刻提高对层序遍历的警觉。

## 它和 DFS 的区别

DFS 更像深入一条分支。

层序遍历则强调同一层的节点先聚在一起看。

所以我现在会这样区分：

- 题目关心层级结构，优先想层序遍历 / BFS
- 题目关心子树递归定义，优先想 DFS

这一步区分特别重要，因为很多树题错的不是代码，而是第一步方向选错。

## 易错点

- 容易把层序遍历硬写成 DFS，结果每层信息维护得很别扭
- 容易知道用队列，却不知道如何划分“当前层”
- 容易漏掉空树这个边界
- 容易把“层序遍历”和“普通 BFS”分开背，没看到它们本来就是一回事

## 我的理解

层序遍历最值得记住的不是模板，而是：

它是在按层组织节点，而不是按递归深度自然流过。

只要题目关心“每一层”，层序遍历通常就是最顺手的入口。
