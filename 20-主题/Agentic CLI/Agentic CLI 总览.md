# Agentic CLI 总览

## 这是什么

`Agentic CLI` 指的是把大模型、工具调用、会话状态、权限控制和任务执行结合在终端里的系统。

它和普通聊天机器人最大的不同，是它不是只“回答”，而是能在受控边界内实际做事。

这组主题笔记的视角，默认以 `Claude Code` 的源码实现为锚点。
如果后面知识库里出现更抽象的 agent 框架总论，这里就保留“实现启发”和“源码证据”，不重复写通用定义。

## 为什么重要

- 它正在重写“命令行工具”和“AI 助手”之间的边界
- 它让模型从信息提供者变成任务执行者
- 它天然适合软件开发、运维、研究、自动化这些终端场景

## 我研究这类系统时会先看什么

如果我要判断一个 agentic CLI 是否成熟，我通常先看五件事：

1. 用户入口长什么样
2. 主循环怎么推进任务
3. 工具系统怎么定义能力
4. 权限系统怎么管边界
5. 扩展机制怎么接外部能力

这五件事基本就能看出这个系统只是“会调模型”，还是已经开始像一个真正的产品。

## 核心组成

- 用户入口：CLI 参数、交互 REPL、slash command、IDE bridge
- 推理与编排：对话主循环、工具调用循环、上下文拼接、预算控制
- 能力层：文件、搜索、Shell、Web、MCP、LSP、任务、子代理
- 治理层：权限、策略、设置、日志、遥测、持久化
- 扩展层：技能、插件、外部资源、团队/多代理

## 我从 Claude Code 里学到的结构

- 用 `commands` 管“用户主动入口”
- 用 `tools` 管“模型被动可调能力”
- 用 `QueryEngine` 管“一轮轮任务推进”
- 用 `permissions` 管“能不能做”
- 用 `skills / plugins / MCP` 管“如何扩展”

## 我的理解

一个成熟的 agentic CLI，真正难的不是“接上模型”，而是把执行力、可控性、可扩展性、可恢复性同时做出来。
