# Agentic CLI 研究路线

## 这篇笔记要解决什么

这篇不讲某个单点概念，而是记录我研究 agentic CLI 时的观察顺序和拆解路线。

## 我会按什么顺序研究

### 1. 先看入口

要先看系统怎么接收任务：

- CLI 参数
- REPL
- slash command
- IDE / bridge

### 2. 再看主循环

最关键的问题是：

- 用户输入怎么进入系统
- 一轮推理怎么触发
- 工具结果怎么回到下一轮推理
- 状态如何保存

### 3. 再看能力层

这里主要看：

- tools
- commands
- 文件、Shell、Web、搜索
- MCP、LSP、Task、子代理

### 4. 再看治理层

也就是：

- 权限
- 设置
- policy
- 日志
- 持久化

### 5. 最后看扩展层

包括：

- skills
- plugins
- MCP
- 多代理

## 为什么按这个顺序

因为如果一开始就钻进工具实现细节，很容易只看到局部能力，却抓不到系统骨架。

先看入口和主循环，才能理解后面每个模块为什么存在。
