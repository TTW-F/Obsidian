# Git 与 GitHub 总览

## 这是什么

这页是 Git 与 GitHub 相关笔记的总入口。

Git 负责本地版本控制模型，GitHub 则把协作、托管和代码审阅流程放到远程平台上。两者经常一起出现，但记忆时最好区分“本地版本控制”与“远程协作平台”这两层。

## 为什么重要

- Git 是开发协作的基础设施，很多问题都和工作区、暂存区、提交历史、分支和合并有关
- GitHub 把 Pull Request、远程仓库、协作审阅和自动化流程接进日常开发
- 如果总入口不清楚，后续笔记容易在命令、概念和流程之间来回跳

## 易错点

- 容易把 Git 命令背成清单，却不理解工作区、暂存区和提交历史之间的关系
- 容易把 Git 和 GitHub 当成同一个东西，结果概念边界越来越乱
- 容易把排错经验只放进命令笔记里，而没有回收到流程或概念笔记
