# AI工程总览

## 这是什么

这一区域用来沉淀和 AI 应用开发相关、而且会长期反复复用的工程知识。

我现在把它理解成一个偏“工程实现”的领域入口，重点关注 agent 运行时、模型接入、工具编排、状态管理、Tracing、评估，以及工作区能力这类主题。

## 为什么重要

- 这类知识通常不是一次性项目经验，而是会反复出现在不同产品和框架里
- 如果没有一个稳定领域入口，后续关于 SDK、运行时和工程能力的笔记容易分散
- 它可以帮助我区分“长期工程知识”和“某个主题下的专题研究”

## 适合继续放进来的主题

- Agent SDK / 框架
- Prompt 与上下文工程

- Session / Memory / 状态恢复
- Tracing / 评估 / 调试
- Sandbox / Realtime / Voice

## 我会怎么使用这一区域

- 需要找长期稳定的工程知识时，先从这里进入
- 某个主题如果已经从单点笔记长成系列内容，再决定是否进入这里
- 主题区的研究笔记成熟后，也可以回流到这个领域做总入口

## 易错点

- 容易把一次性调研结论直接放进领域区，结果长期维护成本很高
- 容易把“产品总览”和“主题研究”混在一起，导致入口模糊
- 容易只列主题清单，不说明这个领域到底覆盖什么边界
